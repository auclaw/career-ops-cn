# 模式：流水线 — URL 收件箱（第二大脑）

处理累积在 `data/pipeline.md` 中的职位 URL。用户随时添加 URL，然后执行 `/career-ops pipeline` 来处理全部。

## 工作流

1. **读取** `data/pipeline.md` → 在"待处理"部分查找 `- [ ]` 项目
2. **对每个待处理 URL**：
   a. 计算下一个顺序 `报告编号`（读取 `reports/`，取最大数字 + 1）
   b. **提取 JD** 使用 Playwright（browser_navigate + browser_snapshot）→ WebFetch → 浏览器搜索
   c. 如果 URL 不可访问 → 标记为 `- [!]` 加备注并继续
   d. **执行完整自动流水线**：A-F 评估 → 报告 .md → PDF（如果分数 >= 3.0）→ 追踪器
   e. **从"待处理"移动到"已处理"**：`- [x] #NNN | URL | 公司 | 职位 | Score/5 | PDF ✅/❌`
3. **如果有 3+ 待处理 URL**，并行启动代理（Agent 工具使用 `run_in_background`）以最大化速度。
4. **完成后**，显示汇总表：

```
| # | 公司 | 职位 | 分数 | PDF | 推荐行动 |
```

## pipeline.md 格式

```markdown
## 待处理
- [ ] https://jobs.example.com/posting/123
- [ ] https://boards.greenhouse.io/company/jobs/456 | Company Inc | 高级产品经理
- [!] https://private.url/job — 错误：需要登录

## 已处理
- [x] #143 | https://jobs.example.com/posting/789 | Acme Corp | AI 产品经理 | 4.2/5 | PDF ✅
- [x] #144 | https://boards.greenhouse.io/xyz/jobs/012 | BigCo | 解决方案架构师 | 2.1/5 | PDF ❌
```

## 从 URL 智能检测 JD

1. **Playwright（首选）：** `browser_navigate` + `browser_snapshot`。适用于所有 SPA。
2. **WebFetch（后备）：** 用于静态页面或 Playwright 不可用时。
3. **浏览器搜索（最后手段）：** 在索引 JD 的二级门户搜索。

**特殊情况：**
- **LinkedIn：** 可能需要登录 → 标记 `[!]` 请用户粘贴文本
- **PDF：** 如果 URL 指向 PDF，直接用 Read 工具读取
- **`local:` 前缀：** 读取本地文件。例如：`local:jds/linkedin-pm-ai.md` → 读取 `jds/linkedin-pm-ai.md`

## 自动编号

1. 列出 `reports/` 中所有文件
2. 从前缀提取数字（例如，`142-medispend...` → 142）
3. 新数字 = 找到的最大值 + 1

## 数据源同步

处理任何 URL 之前，验证同步：
```bash
node cv-sync-check.mjs
```
如果不同步，继续前警告用户。
